# ADR-0001: Goal and Scope

## Context
서비스 운영에서 중요한 것은 “한 번 띄우기”가 아니라, 재현 가능한 배포와 운영 절차(모니터링, 롤백, 장애 대응, 비용/보안 통제)이다.
이를 문서와 자동화 가능한 형태로 남겨, 동일한 환경을 다시 만들고 운영할 수 있게 한다.

## Decision
본 레포는 “운영 가능한 AWS 인프라 블루프린트” 형태로 진행한다.
- 운영 문서(ADR/Runbook/Postmortem)를 핵심 산출물로 관리한다.
- 모든 변경은 `feature/*` 브랜치에서 작업하고 PR로 merge 한다.
- 비용/보안 가드레일을 우선 적용하고, 실습 리소스는 태그 기반으로 관리한다.

## Alternatives
1) 단순 실습 기록(블로그/노션)만 남기기
2) 애플리케이션 기능 개발에 집중하기(인프라/운영 문서 최소화)
3) 단일 배포 성공을 목표로 하고 운영(모니터링/롤백/정리)을 생략하기

## Consequences
- 장점: 운영 관점의 산출물(문서/절차/체크리스트/자동화)을 통해 재현성과 신뢰도를 높일 수 있다.
- 단점: 기능 개발 결과물의 양은 상대적으로 적을 수 있다.
- 리스크: 실습 비용 발생 가능 → 비용 가드레일 및 정리 절차로 통제한다.

## Status
Accepted
